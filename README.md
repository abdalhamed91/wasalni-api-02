# وصلني API — خادم Node/Express

خادم REST لتطبيق «وصلني» لمشاركة الرحلات. مبني على **Express 5** و**SQLite** المدمج في Node 22 (`node:sqlite`) و**JWT** للمصادقة.

## المتطلبات
- Node.js **الإصدار 22 أو أحدث** (ضروري — يستخدم `node:sqlite` المدمج).

## التشغيل
```bash
cd wasalni-api
npm install
node server.js
```
يعمل الخادم على `http://localhost:4000/api`. تُنشأ قاعدة البيانات `wasalni.db` تلقائياً في أول تشغيل (مع بذور عروض الرحلات).

### متغيّرات البيئة (اختيارية)
| المتغيّر | الافتراضي | الغرض |
|---|---|---|
| `PORT` | `4000` | منفذ الخادم |
| `JWT_SECRET` | سرّ تطوير | **غيّره في الإنتاج** |
| `DB_PATH` | `./wasalni.db` | مسار قاعدة البيانات |
| `NODE_ENV` | — | اضبطه `production` لإيقاف وضع التطوير |

## ربط التطبيق بالخادم
في مجلد التطبيق `wasalni-app`، شغّل Expo مع تمرير عنوان الخادم:
```bash
# متصفح / iOS محاكي:
EXPO_PUBLIC_API_URL=http://localhost:4000 npx expo start

# محاكي أندرويد (يصل للمضيف عبر 10.0.2.2):
EXPO_PUBLIC_API_URL=http://10.0.2.2:4000 npx expo start
```
عند **عدم** تمرير المتغيّر، يعمل التطبيق بالكامل في **الوضع المحلي** (بلا خادم) — وهذه هي النسخة الحية المرفقة `wasalni-web-live`.

## المصادقة (وضع التطوير)
- إرسال الرمز يعيد `devCode` في الاستجابة (يُعرض تلقائياً في شاشة الرمز) — لا حاجة لبوابة SMS أثناء التطوير.
- الرمز السرّي `0000` يعمل دائماً في وضع التطوير لأي رقم.
- في الإنتاج: اربط بوابة SMS في `src/auth.js → sendOtp` واضبط `NODE_ENV=production`.

## نظرة على النقاط
المصادقة: `POST /auth/otp/send`, `POST /auth/otp/verify`
الملف: `GET/PATCH /me`, `PUT /me/vehicle`
المحفظة: `GET /wallet`, `POST /wallet/topup`
الأرباح: `GET /earnings`, `POST /earnings/withdraw`
السائق: `GET/POST /trips`, `POST /trips/:id/{cancel|start|complete}`, `POST /requests/:id/{accept|reject}`
الراكب: `GET /rides/search`, `GET/POST /bookings`, `POST /bookings/:id/status`
أخرى: `GET /threads`, `POST /threads/:id/messages`, `GET /notifications`, `GET/POST /places`, `GET/POST/DELETE /cards`

## قواعد العمل المطبّقة في الخادم
- **عمولة المنصة 15%** تُخصم من إجمالي أجرة الرحلة المكتملة، والصافي يُضاف لأرباح السائق.
- **حراسة الدول**: طلب الرحلات متاح فقط في السعودية والأردن ومصر (`SERVICE_COUNTRIES`).
- التحقق من الرصيد قبل الحجز، واسترجاع المبلغ عند الإلغاء، وحدود المقاعد حسب سعة المركبة.
- عزل كامل: كل مستخدم يصل لبياناته فقط (تحقق ملكية على كل عملية).

## لوحة الإدارة (Admin Dashboard)
ملف `admin.html` لوحة تحكم كاملة مربوطة بالخادم مباشرة.

التشغيل:
1. شغّل الخادم: `node server.js`
2. افتح `admin.html` في المتصفح (أو قدّمه عبر أي خادم ثابت).
3. عنوان الخادم الافتراضي: `http://localhost:4000` — كلمة مرور الإدارة: `wasalni-admin` (غيّرها عبر `ADMIN_SECRET`).

الصفحات: لوحة معلومات حيّة (مستخدمون/سائقون/رحلات/إيراد/عمولة)، إدارة المستخدمين (إيقاف/تفعيل)، السائقون ومركباتهم، توثيق السائقين (اعتماد/رفض)، الرحلات، الحجوزات، المالية (دفتر العمولات)، والإعدادات (تعديل نسبة العمولة والدول المُفعّلة). كل البيانات حقيقية من قاعدة البيانات، مع وضع ليلي.

نقاط الإدارة على `/api/admin/*`: `login, stats, users, users/:id/status, drivers/pending, drivers/:id/verify, trips, bookings, finance, settings`.


## ميزات اللوحة المتقدّمة (جديد)
- **الرسوم البيانية**: لوحة المعلومات تعرض الإيراد والحجوزات لآخر 7 أيام، وتوزيع المستخدمين حسب الدولة، وحالات الرحلات.
- **بحث وفلترة**: شريط بحث حيّ فوق كل جدول (مستخدمون/رحلات/حجوزات/مالية).
- **تصدير CSV**: زر تصدير لكل جدول (يدعم العربية عبر BOM).
- **الدول والتسعير**: صفحة لكل دولة تتحكّم بـ:
  - تفعيل/تعطيل الخدمة.
  - **نوع ربح المنصة**: نسبة مئوية أو **مبلغ مقطوع لكل رحلة** — مستقل لكل دولة.
  - **تسعير المقعد بالكيلومتر**: سعر/كم + سقف أعلى، بعملة كل دولة.
- التطبيق يقترح سعر المقعد تلقائيًا حسب المسافة وتسعير دولة السائق (زر «اقترح السعر حسب المسافة» في نشر المسار).

### نقاط API الإدارية الجديدة
`GET /api/admin/countries` · `PATCH /api/admin/countries/:code` · `GET /api/admin/charts` · `GET /api/admin/export/:kind` (users|trips|bookings|finance)
ونقطة التطبيق: `GET /api/pricing/quote?km=<distance>`
